WaveBoard API
Session key management for the WaveBoard desktop app. Vercel serverless + MongoDB Atlas, authenticated via Discord bot.
Base URLapi-waveboard.vercel.app
DatabaseMongoDB Atlas
RuntimeVercel Serverless
Authx-bot-secret
Overview
Public Endpoints
/api/ping  /api/verify
Bot-Only Endpoints
/api/register  /api/grant  /api/revoke  /api/sessions
Session Key Format
SK-[12chars]-[year]
Plans
free  |  pro
💡 Bot-only endpoints require the x-bot-secret header matching your BOT_SECRET env var. Only your Discord bot should call these routes — end users authenticate via /api/verify with their session key.

Environment Variables

VariableTypeDescription
MONGODB_URIstringMongoDB Atlas connection string
MONGODB_DBstringDatabase name — defaults to waveboard
BOT_SECRETstringShared secret between this API and your Discord bot
Public Endpoints
GET /api/ping Public ▶

Health check. Returns API status and current timestamp. Use this to confirm the deployment is live.

Response 200

json
{
  "ok":        true,
  "version":   "1.0.0",
  "status":    "online",
  "timestamp": "2025-01-01T00:00:00.000Z"
}
GET /api/verify Public ▶

Verify a session key and return the associated user. Called by the WaveBoard desktop app on startup. Also updates the user's lastAccess timestamp.

Query Parameters

ParamTypeDescription
keyrequiredstringSession key — e.g. SK-AbCd1234-2025

Response 200

json
{
  "ok": true,
  "data": {
    "key":         "SK-AbCd1234-2025",
    "displayName": "StreamerName",
    "discordId":   "123456789012345678",
    "plan":        "free",
    "createdAt":   "2025-01-01T00:00:00.000Z"
  }
}

Response 404

json
{ "ok": false, "error": "Session not found. Check your key or register via Discord." }
Bot-Only Endpoints
🔒 These endpoints require the x-bot-secret header. Called exclusively by the Discord bot — never expose this secret to end users.
POST /api/register Bot Only ▶

Register a new user and generate their session key. Triggered when a user runs /register in Discord. If already registered, returns their existing session without creating a duplicate.

Headers

HeaderTypeDescription
x-bot-secretrequiredstringMust match BOT_SECRET env var
Content-Typeoptionalstringapplication/json

Request Body

FieldTypeDescription
discordIdrequiredstringUser's Discord snowflake ID
displayNamerequiredstringUser's Discord display name

Response 201 — New user created

json
{
  "ok": true,
  "data": {
    "key":         "SK-AbCd1234-2025",
    "displayName": "StreamerName",
    "discordId":   "123456789012345678",
    "plan":        "free",
    "createdAt":   "2025-01-01T00:00:00.000Z"
  }
}

Response 200 — Already registered

json
{ "ok": true, "alreadyExists": true, "data": { /* same as 201 */ } }
POST /api/grant Bot Only ▶

Update a user's plan. Triggered by /grant @user pro or /grant @user free in Discord.

Request Body

FieldTypeDescription
discordIdrequiredstringTarget user's Discord ID
planrequiredfree | proPlan to assign

Response 200

json
{ "ok": true, "message": "Plan updated to pro" }

Response 404

json
{ "ok": false, "error": "User not found" }
POST /api/revoke Bot Only ▶

Permanently delete a user's session. Triggered by /revoke @user in Discord. The user will need to run /register again to get a new key.

Request Body

FieldTypeDescription
discordIdrequiredstringTarget user's Discord ID

Response 200

json
{ "ok": true, "message": "Session revoked" }
GET /api/sessions Bot Only ▶

List all registered sessions sorted by creation date. Triggered by /sessions in Discord.

Response 200

json
{
  "ok":    true,
  "count": 3,
  "data": [
    {
      "key":         "SK-AbCd1234-2025",
      "displayName": "StreamerName",
      "plan":        "pro",
      "createdAt":   "2025-01-01T00:00:00.000Z",
      "lastAccess":  "2025-06-01T12:00:00.000Z"
    }
  ]
}
Error Codes
StatusMeaning
400Bad request — missing or invalid fields in body or query params
401Unauthorized — missing or incorrect x-bot-secret header
404Not found — session or user does not exist
405Method not allowed — wrong HTTP method for this route
500Server error — check MongoDB connection string and Vercel logs
Deployment
1

Install Vercel CLI and deploy

bash
npm install -g vercel
vercel --prod
2

Set environment variables in the Vercel dashboard

env
MONGODB_URI = mongodb+srv://user:pass@cluster.mongodb.net/
MONGODB_DB  = waveboard
BOT_SECRET  = your-random-secret-here
3

Add the same BOT_SECRET to your Discord bot .env

env
BOT_SECRET = your-random-secret-here
API_URL    = https://api-waveboard.vercel.app
4

Verify the deployment is live

bash
curl https://api-waveboard.vercel.app/api/ping
