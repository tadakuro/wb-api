WaveBoard API
Session management for the WaveBoard desktop app. All bot-facing routes are protected and not intended for end users.
Base URLapi-waveboard.vercel.app
DatabaseMongoDB Atlas
RuntimeVercel Serverless
Authx-bot-secret
Overview
💡 Public endpoints are open. Bot-only endpoints require the x-bot-secret header — these are called exclusively by the Discord bot and are not meant for direct use.
Public Endpoints
GET /api/ping Public ▶

Health check. Confirms the API is up and running.

Response 200

json
{
  "ok":        true,
  "version":   "1.0.0",
  "status":    "online",
  "timestamp": "2025-01-01T00:00:00.000Z"
}
GET /api/verify Public ▶

Verifies a session key. Called by the WaveBoard desktop app on startup to authenticate the user.

Query Parameters

ParamTypeDescription
keyrequiredstringThe session key from your Discord DM

Response 200

json
{
  "ok": true,
  "data": {
    "displayName": "StreamerName",
    "discordId":   "123456789012345678",
    "createdAt":   "2025-01-01T00:00:00.000Z"
  }
}

Response 404

json
{ "ok": false, "error": "Session not found. Check your key or register via Discord." }
Bot-Only Endpoints
🔒 These routes are restricted. They require the x-bot-secret header and are only called by the Discord bot internally.
POST /api/register Bot Only ▶

Registers a new user and issues them a session key. Fires when a user runs /register in Discord. Returns the existing session if they're already registered.

Request Body

FieldTypeDescription
discordIdrequiredstringUser's Discord ID
displayNamerequiredstringUser's Discord display name

Response 201 — Created

json
{ "ok": true, "data": { /* session object */ } }

Response 200 — Already registered

json
{ "ok": true, "alreadyExists": true, "data": { /* session object */ } }
POST /api/grant Bot Only ▶

Updates a user's access level. Triggered by an admin via Discord.

Request Body

FieldTypeDescription
discordIdrequiredstringTarget user's Discord ID
planrequiredstringAccess level to assign

Response 200

json
{ "ok": true, "message": "Plan updated" }
POST /api/revoke Bot Only ▶

Removes a user's session entirely. Triggered by an admin via Discord. The user will need to register again to regain access.

Request Body

FieldTypeDescription
discordIdrequiredstringTarget user's Discord ID

Response 200

json
{ "ok": true, "message": "Session revoked" }
GET /api/sessions Bot Only ▶

Returns a list of all registered sessions. Admin use only, triggered via Discord.

Response 200

json
{
  "ok":    true,
  "count": 3,
  "data": [
    {
      "displayName": "StreamerName",
      "createdAt":   "2025-01-01T00:00:00.000Z",
      "lastAccess":  "2025-06-01T12:00:00.000Z"
    }
  ]
}
